# 东软环保公众监督系统前后端接口文档

## 一、接口总说明

本文档用于前端与后端联调，约定四端基础版所需接口。

四端包括：

1. 公众监督员端
2. 网格员端
3. 系统管理员端
4. 决策者端

前端统一通过 Axios 请求后端接口。默认后端接口前缀为：

```txt
/api
```

例如前端调用：

```txt
/api/aqiFeedback/saveAqiFeedback
```

如果后端实际没有 `/api` 前缀，前端只需要修改 Axios 的 `baseURL`。

## 二、统一返回格式

所有接口建议统一返回以下格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

字段说明：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | Number | 状态码，`200` 表示成功 |
| message | String | 提示信息 |
| data | Any | 返回数据，可以是对象、数组或 null |

失败示例：

```json
{
  "code": 400,
  "message": "手机号或密码错误",
  "data": null
}
```

## 三、公众监督员接口

### 1. 公众监督员注册

```txt
POST /api/supervisor/saveSupervisor
```

请求参数：

```json
{
  "phone": "13800138000",
  "password": "123456",
  "realName": "张三"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| phone | String | 是 | 手机号码 |
| password | String | 是 | 登录密码 |
| realName | String | 是 | 真实姓名 |

返回示例：

```json
{
  "code": 200,
  "message": "注册成功",
  "data": {
    "id": "S001",
    "phone": "13800138000",
    "realName": "张三"
  }
}
```

### 2. 公众监督员登录

```txt
POST /api/supervisor/login
```

请求参数：

```json
{
  "phone": "13800138000",

}
```

```json

  "data": {
    "id": "S001",
    "phone": "13800138000",
```

## 四、空气质量反馈接口

### 1. 保存空气质量反馈

```txt
POST /api/aqiFeedback/saveAqiFeedback
```

请求参数：

```json
{
  "supervisorId": "S001",
  "provinceId": "P001",
  "cityId": "C001",
  "estimatedAqi": 86,
  "description": "附近有明显异味，空气质量较差"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| supervisorId | String | 是 | 公众监督员 ID |
| provinceId | String | 是 | 省份 ID |
| cityId | String | 是 | 城市 ID |
| estimatedAqi | Number | 是 | 公众预估 AQI，范围 0-500 |
| description | String | 是 | 反馈描述 |

返回示例：

```json
{
  "code": 200,
  "message": "反馈提交成功",
  "data": {
    "id": "F001",
    "status": 0
  }
}
```

状态说明：

| status | 说明 |
| --- | --- |
| 0 | 未处理 |
| 1 | 已指派 |
| 2 | 已确认 |

### 2. 查询公众监督员历史反馈

```txt
GET /api/aqiFeedback/listAqiFeedbackBySupervisorId?supervisorId=S001
```

返回示例：

```json
{
  "code": 200,
  "message": "success",
  "data": [
    {
      "id": "F001",
      "supervisorId": "S001",
      "provinceId": "P001",
      "provinceName": "北京市",
      "cityId": "C001",
      "cityName": "朝阳区",
      "estimatedAqi": 86,
      "description": "附近有明显异味，空气质量较差",
      "status": 0,
      "createTime": "2026-06-02 10:00:00"
    }
  ]
}
```

### 3. 查询全部反馈

系统管理员端使用。

```txt
GET /api/aqiFeedback/listAqiFeedbackAll
```

可选查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| status | Number | 按状态筛选 |
| provinceId | String | 按省份筛选 |
| cityId | String | 按城市筛选 |

示例：

```txt
GET /api/aqiFeedback/listAqiFeedbackAll?status=0
```

返回数据格式同历史反馈列表。

### 4. 指派网格员

```txt
POST /api/aqiFeedback/updateAqiFeedbackAssign
```

请求参数：

```json
{
  "feedbackId": "F001",
  "gridMemberId": "G001",
  "status": 1
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| feedbackId | String | 是 | 反馈 ID |
| gridMemberId | String | 是 | 被指派的网格员 ID |
| status | Number | 是 | 指派后状态，固定为 1 |

返回示例：

```json
{
  "code": 200,
  "message": "指派成功",
  "data": null
}
```

## 七、AQI 检测确认与统计接口

### 1. 保存网格员 AQI 检测确认信息

```txt
POST /api/statistics/saveStatistics
```

请求参数：

```json
{
  "feedbackId": "F001",
  "gridMemberId": "G001",
  "so2": 20.5,
  "co": 0.8,
  "pm25": 68.2,
  "aqi": 95,
  "confirmTime": "2026-06-02 11:00:00"
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| feedbackId | String | 是 | 反馈 ID |
| gridMemberId | String | 是 | 网格员 ID |
| so2 | Number | 是 | SO2 浓度 |
| co | Number | 是 | CO 浓度 |
| pm25 | Number | 是 | PM2.5 浓度 |
| aqi | Number | 是 | 实测 AQI |
| confirmTime | String | 否 | 确认时间，可由后端生成 |

返回示例：

```json

  "code": 200,

}
```

## 九、AQI 等级规则

前端页面会根据 AQI 数值展示等级和颜色，建议前后端保持一致。

| AQI 范围 | 等级 | 前端颜色建议 |
| --- | --- | --- |
| 0-50 | 优 | 绿色 |
| 51-100 | 良 | 黄色 |
| 101-150 | 轻度污染 | 橙色 |
| 151-200 | 中度污染 | 红色 |
| 201-300 | 重度污染 | 紫色 |
| 301-500 | 严重污染 | 深紫色 |

## 十、前端接口文件规划

后续前端将按以下方式封装接口：

| 文件 | 接口函数示例 |
| --- | --- |
| `src/api/supervisor.js` | `loginSupervisor`、`saveSupervisor`、`getSupervisorById` |
| `src/api/aqiFeedback.js` | `saveAqiFeedback`、`listAqiFeedbackAll`、`assignAqiFeedback` |
| `src/api/gridMember.js` | `loginGridMember`、`listGridMemberByProvinceId` |
| `src/api/statistics.js` | `saveStatistics`、`listProvinceItemTotalStatis`、`listAqiDistributeStatis` |
| `src/api/admins.js` | `loginAdmin` |

页面只调用接口函数，不直接拼接请求路径。

## 十一、联调注意事项

1. 后端字段命名尽量与本文档保持一致。
2. 如果后端使用不同字段名，前端可以在 `api` 层做转换。
3. 登录成功后，前端会保存用户信息和角色信息。
4. 列表接口建议返回数组，即使没有数据也返回空数组 `[]`。
5. 提交接口成功后可以返回新增数据 ID，方便前端跳转详情或刷新列表。
6. 所有时间字段建议使用统一格式：`YYYY-MM-DD HH:mm:ss`。
7. 所有金额、浓度、AQI 等数值字段建议返回 Number 类型，不要返回字符串。
